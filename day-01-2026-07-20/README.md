# Ημέρα 1 — Τα LLMs βλέπουν *tokens*, όχι λέξεις

**Ημερομηνία:** 20 Ιουλίου 2026
**Θέμα:** Πρώτη επαφή με Python + η βασικότερη έννοια των LLMs: το token

---

## Γιατί ξεκινάμε από εδώ

Κάθε φορά που στέλνεις κείμενο σε ένα μοντέλο (ChatGPT, Claude, Gemini), το μοντέλο **δεν βλέπει λέξεις**. Το κείμενο κόβεται πρώτα σε μικρά κομμάτια που λέγονται **tokens**. Ένα token είναι περίπου 4 χαρακτήρες στα αγγλικά — κάτι σαν συλλαβή.

Παράδειγμα: η λέξη `unbelievable` μπορεί να γίνει `un` + `bel` + `iev` + `able` = 4 tokens.

Αυτό δεν είναι θεωρητική λεπτομέρεια. Είναι **η μονάδα χρέωσης**. Οι εταιρείες πληρώνουν ανά 1.000.000 tokens. Επίσης κάθε μοντέλο έχει όριο (context window) — π.χ. 200.000 tokens. Αν δεν ξέρεις να μετράς tokens, δεν μπορείς να υπολογίσεις ούτε κόστος ούτε αν χωράει το κείμενό σου.

Σήμερα θα γράψεις τα πρώτα σου προγράμματα Python και θα μετρήσεις tokens μόνος σου.

## Έννοιες που θα συναντήσεις σήμερα

**Μεταβλητή (variable)** — ένα κουτί με όνομα όπου βάζεις μια τιμή:
```python
text = "Γεια σου"
```
Το `text` είναι το όνομα, το `"Γεια σου"` η τιμή. Τα εισαγωγικά σημαίνουν "αυτό είναι κείμενο" (string).

**Συνάρτηση (function)** — μια συνταγή που παίρνει κάτι, κάνει δουλειά, και επιστρέφει αποτέλεσμα:
```python
def count_chars(text):
    return len(text)
```
`def` = "όρισε συνάρτηση". `return` = "δώσε πίσω αυτό". Την καλείς γράφοντας `count_chars("γεια")` → σου δίνει `4`.

**Λίστα (list)** — πολλά πράγματα στη σειρά, μέσα σε αγκύλες:
```python
prompts = ["Γεια", "Τι κάνεις;", "Πες μου ένα αστείο"]
```

**Loop (βρόχος)** — "κάνε κάτι για κάθε στοιχείο":
```python
for p in prompts:
    print(p)
```

**Dictionary (dict)** — ζευγάρια κλειδί→τιμή, σαν λεξικό:
```python
prices = {"gpt-4o": 2.50, "claude-sonnet": 3.00}
prices["gpt-4o"]   # → 2.50
```

## Task A — Μετρητής tokens με τον κανόνα του 4 (~30 λεπτά)

**Στόχος:** ένα πρόγραμμα που παίρνει κείμενο και σου λέει πόσοι χαρακτήρες, πόσες λέξεις και περίπου πόσα tokens είναι.

**Αρχείο:** `task-a/token_counter.py`

**Βήματα:**

1. Αντίγραψε το `task-a/token_counter.py` στον φάκελό σου.
2. Άνοιξέ το με έναν editor (VS Code, ή `nano token_counter.py`).
3. Συμπλήρωσε ένα-ένα τα `# TODO:`. Μην πας στο επόμενο πριν τρέξει το προηγούμενο.
4. Τρέξε το:
   ```bash
   python3 token_counter.py
   ```

**Πώς ξέρεις ότι δουλεύει** — πρέπει να δεις κάτι σαν:
```
Κείμενο: Explain what a large language model is.
  Χαρακτήρες: 39
  Λέξεις: 7
  Tokens (εκτίμηση): 10
```

**Χρήσιμα:**
- `len("abc")` → `3` (μήκος)
- `"a b c".split()` → `["a", "b", "c"]` (σπάει σε λέξεις)
- `round(10 / 4)` → `2` (στρογγυλοποίηση)
- `print(f"Λέξεις: {n}")` → f-string, βάζει την τιμή του `n` μέσα στο κείμενο

## Task B — Υπολογιστής κόστους API (~1 ώρα)

**Στόχος:** ένα πρόγραμμα που, δεδομένου ενός prompt και ενός μοντέλου, σου λέει πόσο θα κοστίσει η κλήση — και τι θα κόστιζε αν την έκανες 10.000 φορές τον μήνα.

Αυτό είναι το είδος εργαλείου που φτιάχνει κανείς **την πρώτη εβδομάδα σε δουλειά με LLMs**. Το κόστος είναι το πρώτο πράγμα που ρωτάει ο manager.

**Αρχείο:** `task-b/cost_calculator.py`

**Βήματα:**

1. Θα χρησιμοποιήσεις ένα dict με τιμές μοντέλων (στο αρχείο, με τιμές σε δολάρια ανά 1 εκατομμύριο tokens).
2. Συμπλήρωσε τα `# TODO:`.
3. Τρέξε το:
   ```bash
   python3 cost_calculator.py
   ```

**Πώς ξέρεις ότι δουλεύει** — πρέπει να δεις πίνακα σαν:
```
=== Υπολογισμός κόστους ===
Prompt: 63 tokens | Απάντηση: 500 tokens

gpt-4o-mini      $0.000309 ανά κλήση  |  $3.09 για 10.000 κλήσεις
gpt-4o           $0.005157 ανά κλήση  |  $51.57 για 10.000 κλήσεις
claude-sonnet-4  $0.007689 ανά κλήση  |  $76.89 για 10.000 κλήσεις
claude-haiku-4   $0.002050 ανά κλήση  |  $20.50 για 10.000 κλήσεις
```

**Η βασική φόρμουλα:**
```
κόστος = (input_tokens / 1_000_000) * τιμή_input
       + (output_tokens / 1_000_000) * τιμή_output
```

**Το σημαντικό συμπέρασμα:** κοίτα τη διαφορά ανάμεσα στο πιο φθηνό και το πιο ακριβό μοντέλο. Είναι σχεδόν 25x. Γι' αυτό η επιλογή μοντέλου είναι μηχανική απόφαση, όχι απλά "πάρε το καλύτερο".
